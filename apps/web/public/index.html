✓ Live  ·  Supabase + Next.js
Real backend. Real data. Real risk engine.

Phase 1 replaces the demo prototype with a production Supabase database. The 5-condition risk algorithm runs server-side. Every stage transition is gate-validated. Clients are real, workflows are persistent, and the CRA deadline clock never stops.

Supabase PostgreSQL backend — 14 migration files, full schema for firms, clients, workflows, stages, tasks, documents and events
5-condition At Risk engine — C1 timeline breach, C2 deadline proximity, C3 document blocker, C4 stage stall, C5 risk history — per workflow, not per client
Gate-enforced stage transitions — Stage 3 hard-blocked until all docs received. Filing blocked until review approved. Hard stops with specific reasons.
REST API — complete endpoints for clients, workflows, stages, tasks, documents, dashboard, and settings
CRA deadlines native — monthly, quarterly, annual filers. Ontario timezone. No manual configuration.

✓ Live  ·  Supabase Auth + RLS
Multi-user roles. File uploads. T1 & T2 templates.

Phase 2 turns AcctOS into a real team tool. Supabase Auth with Row Level Security isolates every firm's data at the database level. Four user roles with distinct permissions. Stage auto-advance when tasks complete. File uploads via Supabase Storage.

4 user roles — Owner (full access + escalations), Senior Accountant (dual review approval), Accountant (assigned clients), Admin (documents + comms)
Row Level Security — every query scoped to firm_id at the database level. A bug in application code cannot leak data between firms.
Auto-advance stages — when all tasks in a stage are complete and the gate condition passes, the workflow advances automatically
File uploads — clients upload directly to Supabase Storage. Document checklist auto-updates. Stage 2 gate unlocks on receipt.
T1, T2 & Bookkeeping templates — same engine, new configurations. T1 for personal returns, T2 for corporate, monthly bookkeeping cycle.

✓ Live  ·  Resend + pg_cron + R2
Automated reminders. Smart rules. R2 file storage.

Phase 3 makes AcctOS sticky. Transactional email sequences run automatically — Day 3 reminder, Day 6 escalation, deadline alerts. Automation rules engine fires background jobs. Cloudflare R2 replaces Supabase Storage for zero egress cost at scale.

Automated document reminder sequence — Day 3: Reminder #1 to client. Day 6: Reminder #2 + escalate to firm owner. Deadline < 5d: urgent alert.
5 automation rules — auto-create workflows, document reminders, escalation, deadline alerts, overdue flags — all configurable per firm
Cloudflare R2 storage — zero egress fees. 3-tier lifecycle: Standard (0–2yr) → Infrequent Access (2–4yr) → Archive (4–7yr, CRA requirement)
Dashboard intelligence — priority suggestions, GST anomaly detection (≥20% quarter-on-quarter change), risk trend per client
pg_cron background jobs — runs inside Supabase Postgres. No external queue until 50+ firms justify complexity. Zero added infra cost.

✓ Live  ·  QBO + Stripe + Client Portal
QuickBooks sync. Client portal. Billing triggers.

Phase 4 plugs AcctOS into real firm workflows. When QuickBooks Online marks a period reconciled, Stage 1 advances automatically — no manual confirmation needed. Clients upload documents through a tokenised portal. Filing completion triggers a Stripe invoice.

QuickBooks Online OAuth — reconciliation webhook auto-advances Stage 1 gate. Zoho Books supported as alternative.
Client portal (no login required) — tokenised URL sent to client. They see pending docs, upload directly, and the checklist auto-updates in AcctOS.
Stripe billing — Starter $49/mo · Growth $149/mo · Scale $299/mo (all CAD, firm-flat). Stage 6 completion triggers invoice automatically.
Payroll Remittances template — monthly and bi-weekly cycles. CRA payroll deadlines built in. Same engine, new configuration.
Full webhook suite — QBO reconciliation events, Zoho Books events, Stripe subscription lifecycle, Resend delivery events.

📅
CRA deadlines native
Monthly, quarterly, and annual CRA deadlines built in. Ontario timezone. No manual configuration. Monthly filers: last day of following month. Quarterly: last day of month after quarter-end.
🔒
Gate enforcement
Stage 3 cannot begin until all documents are received. Filing is blocked until review is approved. Hard stops with specific reasons — not just visual warnings, but database-level prevention.
⚡
5-condition risk engine
C1 timeline breach · C2 deadline proximity · C3 document blocker · C4 stage stall · C5 risk history. Evaluated per workflow. Status is computed dynamically — never stored, never stale.
🏢
Corp vs sole prop branching
Corporation: ITC reconciliation added to Stage 3. Dual review gate if GST > $10,000. Sole proprietor: simplified checklist, revenue threshold check, no ITCs required.
📧
Automated escalation
Document requests go out automatically. Day 3: Reminder #1. Day 6: Reminder #2 + owner escalation. Deadline < 5 days: urgent alert. Full email log per client, per filing.
🛡️
Row Level Security
Every database query scoped to firm_id at the Postgres level via Supabase RLS. A bug in application code cannot leak data between firms. PIPEDA-compliant. Data in ca-central-1.
